# 引用《开发指导.md》

## 步骤 1：调整 Maven 测试依赖与测试插件

### 改动位置

`pom.xml`

### 改动内容

#### 1.1 在 `properties` 中增加 JUnit5 版本号

```xml
<properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    <maven.compiler.encoding>UTF-8</maven.compiler.encoding>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <junit.jupiter.version>5.10.2</junit.jupiter.version>
    <junit.platform.version>1.10.2</junit.platform.version>
</properties>
```

#### 1.2 删除 JUnit4 依赖，替换为 JUnit5 依赖

删除：

```xml
<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.12</version>
    <optional>true</optional>
</dependency>
```

新增：

```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>${junit.jupiter.version}</version>
    <optional>true</optional>
</dependency>
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>${junit.jupiter.version}</version>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.junit.platform</groupId>
    <artifactId>junit-platform-suite</artifactId>
    <version>${junit.platform.version}</version>
    <scope>test</scope>
</dependency>
```

#### 1.3 在 `build.plugins` 中增加 Surefire

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.2.5</version>
    <configuration>
        <useModulePath>false</useModulePath>
    </configuration>
</plugin>
```

## 步骤 2：将 `ApplicaiontContextRunner` 改为 JUnit5 扩展

### 改动位置

`src/main/java/cc/jfire/jfire/helpjunit/ApplicaiontContextRunner.java`

### 改动内容

#### 2.1 替换类继承关系

删除 JUnit4 Runner 继承方式：

```java
public class ApplicaiontContextRunner extends BlockJUnit4ClassRunner
```

改为 JUnit5 扩展接口实现：

```java
public class ApplicaiontContextRunner implements TestInstanceFactory, ParameterResolver
```

#### 2.2 删除 JUnit4 Runner 相关导入

删除：


新增：

```java

import org.junit.jupiter.api.extension.ParameterResolver;
import org.junit.jupiter.api.extension.TestInstanceFactory;
import org.junit.jupiter.api.extension.TestInstanceFactoryContext;
```

#### 2.3 删除 JUnit4 生命周期拼装逻辑

整段删除以下成员：

```java
private final Class<?> klass;

public ApplicaiontContextRunner(Class<?> klass) throws InitializationError

    super(klass);
    this.klass = klass;
}

@Override
@SuppressWarnings("deprecation")
protected Statement methodBlock(FrameworkMethod method)
{
    ...
}

private Statement withRules(...)
{
    ...
}

private Statement withTestRules(...)
{
    ...
}

private Statement withMethodRules(...)
{
    ...

{
    return rules(target);

protected Object createTest()

    ApplicationContext context = new DefaultApplicationContext(klass);

```

#### 2.4 新增 JUnit5 下的测试实例创建逻辑

新增核心实现：

```java
private static final ExtensionContext.Namespace NAMESPACE =
        ExtensionContext.Namespace.create(ApplicaiontContextRunner.class);

@Override
public Object createTestInstance(TestInstanceFactoryContext factoryContext, ExtensionContext extensionContext)
{
    Class<?> testClass = factoryContext.getTestClass();
    ApplicationContext context = getOrCreateContext(extensionContext, testClass);
    return context.getBean(testClass);
}

@Override
public boolean supportsParameter(ParameterContext parameterContext, ExtensionContext extensionContext)
{
    return parameterContext.getParameter().getType() == ApplicationContext.class;
}

@Override
public Object resolveParameter(ParameterContext parameterContext, ExtensionContext extensionContext)
{
    return getOrCreateContext(extensionContext, extensionContext.getRequiredTestClass());
}

private ApplicationContext getOrCreateContext(ExtensionContext extensionContext, Class<?> testClass)
{
    return extensionContext.getStore(NAMESPACE)
            .getOrComputeIfAbsent(testClass, key -> new DefaultApplicationContext(testClass), ApplicationContext.class);
}
```

#### 2.5 类最终形态

```java
package cc.jfire.jfire.helpjunit;

import cc.jfire.jfire.core.ApplicationContext;
import cc.jfire.jfire.core.DefaultApplicationContext;
import org.junit.jupiter.api.extension.ExtensionContext;
import org.junit.jupiter.api.extension.ParameterContext;
import org.junit.jupiter.api.extension.ParameterResolver;
import org.junit.jupiter.api.extension.TestInstanceFactory;
import org.junit.jupiter.api.extension.TestInstanceFactoryContext;

public class ApplicaiontContextRunner implements TestInstanceFactory, ParameterResolver
{
    private static final ExtensionContext.Namespace NAMESPACE =
            ExtensionContext.Namespace.create(ApplicaiontContextRunner.class);

    @Override
    public Object createTestInstance(TestInstanceFactoryContext factoryContext, ExtensionContext extensionContext)
    {
        Class<?> testClass = factoryContext.getTestClass();
        ApplicationContext context = getOrCreateContext(extensionContext, testClass);
        return context.getBean(testClass);
    }

    @Override
    public boolean supportsParameter(ParameterContext parameterContext, ExtensionContext extensionContext)
    {
        return parameterContext.getParameter().getType() == ApplicationContext.class;
    }

    @Override
    public Object resolveParameter(ParameterContext parameterContext, ExtensionContext extensionContext)
    {
        return getOrCreateContext(extensionContext, extensionContext.getRequiredTestClass());
    }

    private ApplicationContext getOrCreateContext(ExtensionContext extensionContext, Class<?> testClass)
    {
        return extensionContext.getStore(NAMESPACE)
                .getOrComputeIfAbsent(testClass, key -> new DefaultApplicationContext(testClass), ApplicationContext.class);
    }
}
```

## 步骤 3：新增 JUnit5 组合注解，作为 `ApplicaiontContextRunner` 的推荐入口

### 改动位置

`src/main/java/cc/jfire/jfire/helpjunit/JfireTest.java`

### 改动内容

新增文件，提供面向测试类的统一接入注解：

```java
package cc.jfire.jfire.helpjunit;

import org.junit.jupiter.api.TestInstance;
import org.junit.jupiter.api.extension.ExtendWith;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

import static org.junit.jupiter.api.TestInstance.Lifecycle.PER_METHOD;

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@TestInstance(PER_METHOD)
@ExtendWith(ApplicaiontContextRunner.class)
public @interface JfireTest
{
}
```

该类在执行阶段的要求：

1. 包名与 `ApplicaiontContextRunner` 保持一致，放在 `helpjunit` 包下。
2. 只负责声明扩展和测试实例生命周期，不混入其他业务逻辑。
3. 后续所有需要容器托管测试实例的测试类都通过该注解接入，而不是直接使用 `@ExtendWith`。

## 步骤 5：补充 `ApplicaiontContextRunner` 的 JUnit5 使用验证

### 改动位置

`src/test/java/cc/jfire/jfire/test/function/ApplicaiontContextRunnerTest.java`

### 改动内容

新增专门验证 JUnit5 扩展行为的测试类，要求覆盖“测试实例来自容器”和“测试方法可注入 `ApplicationContext`”两点。

文件内容：

```java
package cc.jfire.jfire.test.function;

import cc.jfire.jfire.core.ApplicationContext;
import cc.jfire.jfire.core.prepare.annotation.ComponentScan;
import cc.jfire.jfire.core.prepare.annotation.configuration.Configuration;
import cc.jfire.jfire.helpjunit.JfireTest;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.assertNotNull;
import static org.junit.jupiter.api.Assertions.assertSame;

@JfireTest
public class ApplicaiontContextRunnerTest
{
    @Test
    void shouldCreateTestInstanceFromApplicationContext(ApplicationContext context)
    {
        ApplicaiontContextRunnerTest bean = context.getBean(ApplicaiontContextRunnerTest.class);
        assertNotNull(bean);
        assertSame(this, bean);
    }

    @Configuration
    @ComponentScan("cc.jfire.jfire.test.function")
    public static class TestConfig
    {
    }
}
```

执行时注意两点：

1. 该测试类本身需要能被 `DefaultApplicationContext` 注册并返回同一个实例。
2. 如果后续发现 `@ComponentScan("cc.jfire.jfire.test.function")` 范围过大导致干扰，可在实现阶段缩小扫描范围，但文档目标不变，仍需验证扩展创建测试实例的核心能力。
